# 小红书笔记编写助手

为 AI SaaS 工具撰写小红书风格的推荐笔记。

## 输入要求

用户需提供以下任一信息：
- 产品官网 URL（Claude 会自动获取产品信息）
- 产品功能介绍文字
- 产品名称 + 核心功能描述

## 工作流程

### 1. 分析产品
- 识别产品核心价值和差异化功能
- 确定目标用户群体
- 找出产品解决的痛点

### 2. 选择参考范例
根据产品特点，从 `references/examples/` 目录读取 1-2 篇相关范例：

| 产品特点 | 推荐范例 |
|---------|---------|
| 网站/页面构建工具 | 250610.md (Wegic), 250712.md (YoinkUI) |
| 笔记/知识管理工具 | 250702.md (Lazy) |
| AI 搜索/比价工具 | 250817.md (Flight Deals) |
| 创意/内容创作工具 | 250820.md (Moises AI) |
| 其他 AI SaaS | 选择功能最相近的 1-2 篇 |

### 3. 撰写笔记
按照下方风格规范撰写。

## 风格规范

### 标题/开头
- 直接点明产品核心价值，用一句话抓住读者
- 示例：「全球首个 AI 网站团队，将你的创意变成收入」
- 示例：「未来你可能再也不会打开去哪儿、飞猪、携程去比价机票了」

### 内容结构
1. **痛点引入**：描述用户面临的问题/旧方式的不便
2. **产品解决方案**：介绍产品如何解决这个问题
3. **个人体验/观点**：加入使用感受或思考
4. **展望/总结**：对未来的期待或一句话总结

### 语气口吻
- 口语化、亲切自然，像朋友聊天
- 常用「你」「我」拉近距离
- 适当使用反问句增强互动感：「这你受得了吗」「这多好啊」
- 敢于表达真实评价，包括不足之处

### Emoji 使用
- 不使用

### 格式特点
- 段落简短，便于手机阅读
- 使用列表展示功能点或使用场景
- 末尾可加关键词标签（如 #AI工具 #效率神器）

## 输出要求

- **字数**：≤800 中文字符
- **格式**：符合小红书阅读习惯（短段落、有分隔）
- **结构**：痛点→方案→体验→总结
- **语气**：口语化、有个人观点
- **话题**：需要有十个话题，其中三个必须是：AI工具，AI，AI创业
